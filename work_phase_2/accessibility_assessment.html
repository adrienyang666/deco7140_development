Accessibility Assessment

Overview

This assessment identifies key accessibility considerations for the proposed website, using the WCAG 2.2 Level AA guidelines. The aim is to ensure the website is inclusive, usable, and accessible to all users, including those with visual, auditory, cognitive, and physical impairments.

Key Accessibility Features

High Contrast & Readable Fonts

All text uses accessible contrast ratios and legible, sans-serif fonts to support users with low vision.

Scalable Text

Font sizes are responsive and allow user zooming without breaking layout.

Keyboard Navigation

Interactive elements such as flip cards and buttons are fully navigable via keyboard (e.g. tab/enter keys).

Multilingual Support

Language toggle allows users to access recipe content in different languages (e.g., English and Spanish).

Transcripts and Captions

All audio or video-based cooking tutorials include written transcripts and captions for hearing-impaired users.

Clear Layout

Content is structured with logical headings, short paragraphs, and clear labels to reduce cognitive load.

Connection to User Research

Accessibility features directly respond to the needs of the user personas. For example, Nai Nai Li benefits from large fonts and audio content. Emilia, an ESL user, benefits from simplified layout and language toggling. Aiden’s event content will be made accessible with visual contrast and alternative text for promotional graphics.

Standards and References

This website follows the WCAG 2.2 Level AA Guidelines published by the W3C Web Accessibility Initiative. These standards ensure compatibility with assistive technologies and inclusive practices across all devices.

Additional guidance is aligned with ethical expectations outlined in the Australian Computing Society’s Code of Ethics.
